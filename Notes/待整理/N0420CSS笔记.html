CSS选择器

CSS选择器：用于“查找”或选取要设置样式的 HTML 元素。
CSS分类：
  • 简单选择器（根据名称、id、类来选取元素）
  • 组合器选择器（根据它们之间的特定关系来选取元素）
  • 伪类选择器（根据特定状态选取元素）
  • 伪元素选择器（选取元素的一部分并设置其样式）
  • 属性选择器（根据属性或属性值来选取元素）

CSS使用

当浏览器读到样式表时，它将根据样式表中的信息来格式化 HTML 文档。
三种插入样式表的方法：
  • 外部CSS
  • 内部CSS
  • 行内CSS
样式表 说明 Tips
外部CSS 通过使用外部样式表，您只需修改一个文件即可改变整个网站的外观！ 每张 HTML 页面必须在 head 部分的 元素内包含对外部样式表文件的引用 外部样式表可以在任何文本编辑器中编写，并且必须以 .css 扩展名保存。 外部 .css 文件不应包含任何 HTML 标签
内部CSS 如果一张 HTML 页面拥有唯一的样式，那么可以使用内部样式表。 内部样式是在 head 部分的 style元素中进行定义
行内CSS 行内样式（也称内联样式）可用于为单个元素应用唯一的样式。 如需使用行内样式，请将 style 属性添加到相关元素。style 属性可包含任何 CSS 属性。 行内样式失去了样式表的许多优点（通过将内容与呈现混合在一起）。请谨慎使用此方法。
注意：如果在不同样式表中为同一选择器（元素）定义了一些属性，则将使用最后读取的样式表中的值。
层叠顺序：行内样式（在 HTML 元素中）>外部和内部样式表（在 head 部分）>浏览器默认样式

CSS backgrounds [背景]

Attribute 说明 实例 Tips
background-color 指定元素的背景色。 
                    body {
                        background-color: lightblue;
                        }
颜色指定："red" "#ff0000" rgba"255,0,0,0.6"
background-image 指定元素的背景图像。 
                    body {
                        background-image: url("paper.gif");
                      }
默认情况下，图像会重复，以覆盖整个元素
可通过repet设置平铺
实际开发常见于logo或装饰性小图片或超大背景图，优点：利用position，非常容易控制图片位置（精灵图也是一种应用场景）
background-repeat 设置如何平铺对象的 background-image 属性。 
                    body
                    {
                        background-image:url('paper.gif');
                        background-repeat:repeat-y;
                    }
repeat 背景图像将向垂直和水平方向重复。这是默认
repeat-x 只有水平位置会重复背景图像
repeat-y 只有垂直位置会重复背景图像
no-repeat background-image 不会重复
inherit 指定 background-repeat 属性设置应该从父元素继承
background-position 用于指定背景图像的位置。 
                    body {
                        background-image: url("/i/photo/tree.png");
                        background-repeat: no-repeat;
                        background-position: right top;
                        margin-right: 200px;
                      }
注意：对于这个工作在Firefox和Opera，background-attachment必须设置为 "fixed（固定）"
属性值：right center x% y% xpos ypos inherit
如果仅指定一个关键字，其他值将会是"center"；仅指定了一个值，其他值将是50％。
background-attachment 指定背景图像是应该滚动还是固定的（不会随页面的其余部分一起滚动） 
                    body {
                        background-image: url("tree.png");
                        background-repeat: no-repeat;
                        background-position: right top;
                        background-attachment: fixed;
                      }
fixed 不会随着页面的滚动而滚动。
scroll 随着页面的滚动而滚动，这是默认的。
local 背景图片会随着元素内容的滚动而滚动
inherit 设置应该从父元素继承。
background 简写属性，缩短代码，指定所有背景属性 
                    body {
                        background: #ffffff url("tree.png") no-repeat right top;
                      }
在使用简写属性时，属性值的顺序为：
  • background-color
  • background-image
  • background-repeat
  • background-attachment
  • background-position
    • [ 色图铺定位 ]
背景色半透明 
                    background: rgba(250, 150, 140, 0.5);
另外opacity也可半透明

CSS Fonts [字体]

5个通用字体族：
  • 衬线字体（Serif）- 在每个字母的边缘都有一个小的笔触。它们营造出一种形式感和优雅感。
  • 无衬线字体（Sans-serif）- 字体线条简洁（没有小笔画）。它们营造出现代而简约的外观。
  • 等宽字体（Monospace）- 这里所有字母都有相同的固定宽度。它们创造出机械式的外观。
  • 草书字体（Cursive）- 模仿了人类的笔迹。
  • 幻想字体（Fantasy）- 是装饰性/俏皮的字体。
Attribute 说明 实例 Tips
font-family 属性规定文本的字体 
                    .p1 {
                        font-family: "Times New Roman", Times, serif;
                      }
应包含多个字体名称作为“后备”系统，以确保浏览器/操作系统之间的最大兼容性
从需要字体开始，通用系列字体结束。字体名称应以，相隔
字体名称不止一个单词，则必须用引号引起来，例如："Times New Roman"。
font-style 主要用于指定斜体文本。 
                    p.normal {
                        font-style: normal;
                      }
此属性可设置三个值：
  • normal - 文字正常显示
  • italic - 文本以斜体显示
  • oblique - 文本为“倾斜”（倾斜与斜体非常相似，但支持较少）
italic 是使用文字的斜体，oblique 是让没有斜体属性的文字倾斜！
font-weight 指定字体的粗细。 
                    p.normal {
                        font-weight: normal;
                      }
normal 默认值。定义标准的字符
bold 定义粗体字符。
bolder 定义更粗的字符。
lighter 定义更细的字符。
x00 400 等同于 normal，而 700 等同于 bold。
font-size 设置文本的大小。可以是绝对或相对大小。
绝对尺寸：
  • 将文本设置为指定大小
  • 不允许用户在所有浏览器中更改文本大小（可访问性不佳）
  • 当输出的物理尺寸已知时，绝对尺寸很有用
相对尺寸：
  • 设置相对于周围元素的大小
  • 允许用户在浏览器中更改文本大小
 
                    body {font-size:100%;}

                    h1 {
                        font-size: 40px;
                      }

                    h1 {
                    font-size: 2.5em; /* 40px/16=2.5em */
                    }

                    
Hello World
0、在所有浏览器的解决方案中，设置 元素的默认字体大小的是百分比 1、使用像素设置文本大小可以完全控制文本大小，仍然可以使用缩放工具来调整整个页面的大小；
2、为了允许用户调整文本大小，使用 em 设置字体大小，W3C 建议使用 em 尺寸单位。
pixels/16=em。
3、响应式字体大小：使用 vw 单位设置文本大小，文本大小将遵循浏览器窗口的大小，1vw = 视口宽度的 1％。
font-variant 指定是否以 small-caps 字体（小型大写字母）显示文本 
                    p.normal {
                        font-variant: normal;
                      }
                      
                      p.small {
                        font-variant: small-caps;
                      }
normal 默认值。浏览器会显示一个标准的字体。
small-caps 浏览器会显示小型大写字母的字体。
inherit 规定应该从父元素继承 font-variant 属性的值。
font 简写属性，缩短代码，指定所有单个字体属性 
                    p.a {
                        font: 20px Arial, sans-serif;
                      }
                      
                      p.b {
                        font: italic small-caps bold 12px/30px Georgia, serif;
                      }
font 属性是以下属性的简写属性[类粗大/行家]：
  • font-style
  • font-variant
  • font-weight
  • font-size/line-height
  • font-family
必须保留size和family属性，否则font将不起作用。

CSS Border[边框]

Attribute 说明 实例 Tips
border-style 定义边框的样式 
                    h1  {
                        border-style: dotted solid double dashed;
                    }
设置border-style属性，其他 CSS 边框属性才会有作用！
border-width 指定四个边框的宽度 
                    p.two {
                        border-style: solid;
                        border-width: 20px 5px; /* 上边框和下边框为 20px，其他边为 5px */
                      }
                      
                      p.three {
                        border-style: solid;
                        border-width: 25px 10px 4px 35px; /* 上边框 25px，右边框 10px，下边框 4px，左边框 35px */
                      }
可以将宽度设置为特定大小（以 px、pt、cm、em 计），也可以使用以下三个预定义值之一：thin、medium 或 thick
border-color 设置四个边框的颜色。 
                    p.one {
                        border-style: solid;
                        border-color: red;
                      }
                      p.one {
                        border-style: solid;
                        border-color: red green blue yellow; /* 上红、右绿、下蓝、左黄 */
                      }
如果未设置 border-color，则它将继承元素的颜色。
border-color 属性可以设置一到四个值（用于上边框、右边框、下边框和左边框）。
border 简写属性,缩减代码。 
                    p {
                        border: 5px solid red;
                      }
                      
                    p {
                    border-bottom: 6px solid red;
                    background-color: lightgrey;
                    }
border 属性是以下各个边框属性的简写属性(无顺序)：
  • border-width
  • border-style（必需）
  • border-color
    • 还可以只为一个边指定所有单个边框属性
    border-radius 向元素添加圆角边框 
                        p {
                        border: 2px solid red;
                        border-radius: 5px;
                      }
    		1、设置圆：先放一个正方形盒子，radius数值为高度或宽度的一般，50%；
    2、设置矩形：设置为高度 height 的一半；
    3、该属性为简写属性，可取四个值：左上角、右上角、右下角、左下角。

    CSS BOX Model [盒子模型]


    • 内容 - 框的内容，其中显示文本和图像。
    • 内边距 - 清除内容周围的区域。内边距是透明的。
    • 边框 - 围绕内边距和内容的边框。
    • 外边距 - 清除边界外的区域。外边距是透明的。
    • 背景应用于由内容和内边距、边框组成的区域。
    • 内边距、边框和外边距都是可选的，默认值是零。但是，许多元素将由用户代理样式表设置外边距和内边距。可以通过将元素的 margin 和 padding 设置为零来覆盖这些浏览器样式。这可以分别进行，也可以使用通用选择器对所有元素进行设置
    • 内边距、边框和外边距可以应用于一个元素的所有边，也可以应用于单独的边。
    • 外边距可以是负值，而且在很多情况下都要使用负值的外边距。

    CSS Display[显示]

    display 属性规定是否/如何显示元素。
    每个 HTML 元素都有一个默认的 display 值，具体取决于它的元素类型。
    大多数元素的默认 display 值为 block 或 inline。

    块级元素（block element）

    • 块级元素总是从新行开始，并占据可用的全部宽度（尽可能向左和向右伸展）。
    • height、width、margin、padding都可以控制
    • 宽度默认为容器（父级元素）的100%
    • 是一个容器及盒子，内可放行内元素和块级元素
    注意：文字类元素内，不可使用块级元素。如：p、h1~h6，否则出问题

    行内元素（inline element）

    内联元素不从新行开始，仅占用所需的宽度。
    • 内联元素不从新行开始，仅占用所需的宽度。
    • height、width直接设置无效
    • 默认宽度是它内容的宽度
    • 行内元素只能容纳文本或其他行内元素
    注意：链接里不能再放链接；特殊情况链接a里可放块级元素，但最好转换块级模式

    行内块元素

    在行内元素中有几个特殊的标签——img input td ，它们同时具有块级元素和行内元素的特点，有些资料称之为行内块元素
    • 和相邻行内元素（行内块）在一行上，中间有空白间隙。一行可以显示多个
    • 默认宽度就是它们本身内容的宽度
    • height、width、margin、padding都可以控制

    元素显示模式的转换

    特殊情况下，需要元素模式的切换，简单理解：一个模式的元素想要另一模式的特性。
    如：增加a链接的触发范围
    • 转化为块级元素 display:block
    • 转化为行内元素 display:inline
    • 转化为行内块元素 display:inline-block

    块级盒子水平居中

    条件：
    1、 盒子指定 width
    2、左右margin设置为： auto

    两种盒子模型类型 （与 margin 无关）

    1、 box-sizing: content box (默认)

    此时，元素的 width = content 宽度
    盒子会膨胀

    2、 box-sizing: border box 【常用】

    此时，元素的width = content + padding + border 的宽度
    盒子不会膨胀

    外边距合并（外边距塌陷）

    取margin较大者，为公用margin标准

    盒子模型布局稳定性

    建议优先级：width > padding > margin

    嵌套块级元素垂直外边距合并

    对于两个嵌套关系的块级元素，若父级元素没有上 内边距及边框，则父元素的上外边框会与子元素的上 外边距发生合并，合并后的外边距为两者中较大者，即使父级元素上外边距为0，也会发生合并。
    解决方案：给父元素设置边框或者内边距

    清除内外边距

    网页元素很多都带有默认的内外边距，而且不同浏览器默认的不一致。因此，在布局前，首先要清除网页的内外边距
    CSS 第一行代码 ↓ 
            * {
            margin:0;
            padding:0;
        }
    注意：行内元素较特殊，为照顾兼容性，只设置左右margin、padding，不设置上下内外边距。但是转化为块级和行内外元素就可以了

    CSS Position [定位]

    定位可以让盒子自由地在某盒子内移动或固定于屏幕某个位置，且可以压住其他盒子。
    定位=定位模式 + 边偏移
    应用于元素的定位方法的类型。一个属性5个值，共同配合5个属性：top、bottom、left、right、z-index（层级关系）
    相关口诀：子绝父相，就近原则

    原因:
    1、为不影响其他盒子，子级不占有位置（脱离标准流）——选择 absolute
    2、父级 需加定位限制 子级 —— relative absolute fixed sticky
    3、布局时，父级要占有位置（不脱离标准流）——选择 relative
    最终： 子 absolute 父 relative

    属性 描述
    static （静态） 默认。
    静态定位的元素不受 top、bottom、left 和 right 属性的影响。
    它始终根据页面的正常流进行定位
    relative 不脱离文档流的布局。只改变自身的位置，在文档流原先的位置留下空白区域
    定位的起始位置为此元素原先在文档流的位置
    只是可以通过4个属性值改变自身的位置：top、bottom、left、right。
    absolute 脱离文档流的布局。遗留下来的空间由后面元素填充
    定位的起始位置为最近一级带有定位的父元素（position不为static，其他均可），否则为body本身。
    总结：起始位置看爸爸，static爸爸看祖宗（body），然后通过4个属性值改变自身的位置：top、bottom、left、right。
    注意：不设置4个位置属性值，起始位置走正常文档流
    fixed (粘性) 相对于视口（viewport，浏览器窗口）进行偏移，即定位基点是浏览器窗口，固定在网页上
    总结：起始位置不看任何人，只看浏览器窗口，然后通过4个属性值改变自身的位置：top、bottom、left、right。
    注意：不设置4个位置属性值，起始位置走正常文档流
    stick 会产生动态效果，很想fixed和relative的结合：一些时候是relative定位（定位基点是自身默认位置），另一些时候自动变成fixed定位（定位基点是视口）
    总结：不设置4个位置属性值，与relative一样。具体规则：页面滚动，父元素脱离窗口时（部分不可见），就跟fixed一样。其他情况跟relative一样。

    定位总结

    定位模式 是否脱标 移动位置 是否常用
    static 否 不能使用边偏移 很少
    relative 否 相对于自身边偏移 常用
    absolute 是 带有定外的最近父级 常用
    fixed 是 浏览器可视区 常用
    sticky 否 浏览器可视区
    固定定位小技巧
    固定到版心右侧 算法：
    1、移动到网页中线 ———— left:50%
    2、再向左移动版心的一半像素 ———— margin-left:-xx px; 
            position:fixed;
        left:50%;
        margin-left:-400px;
    绝对定位盒子居中
    算法： 1、移动到网页中线 ———— left:50%
    2、再向左移动盒子的一半像素 ———— margin-left:xx px;
    3、向下移动网页的一半 ———— top:50%
    4、再向上移动盒子的一半像素 ———— margin-top:-xx px;
    绝对定位特性
    绝对定位和固定定位也和浮动类似。【行内块元素】
    • 1、 行内元素 添加绝对定位或固定定位，可直接设置宽度与高度
    • 2、 块级元素 添加绝对定位或固定定位，不给宽度与高度，默认大小是内容大小
    注意：
    绝对定位和固定定位会压住标准流盒子；
    浮动不会压住标准流盒子。
    定位叠放顺序
    使用 z-index 来控制盒子前后顺序（z轴）
    • 默认是auto。可为负整数、0、正整数，数值越大越靠上
    • 数值相同，后来居上
    • 数字后面不能加单位
    • 只有定位的盒子才有 z-index 属性

    CSS Outline [轮廓]

    轮廓是在元素周围绘制的一条线，在边框之外，以凸显元素。
    注意：轮廓与边框 [不同之处] 在于：轮廓是在元素边框之外绘制的，并且可能与其他内容重叠。同样，轮廓也不是元素尺寸的一部分；元素的总宽度和高度不受轮廓线宽度的影响。
    Attribute 说明 实例 Tips
    outline-style 指定轮廓样式 
                    p.dotted {outline-style: dotted;}
    		注意：除非设置了 outline-style 属性，否则其他轮廓属性（在下一章中将详细介绍）都不会有任何作用！
    dotted - 定义点状的轮廓。
    dashed - 定义虚线的轮廓。
    solid - 定义实线的轮廓。
    double - 定义双线的轮廓。
    groove - 定义 3D 凹槽轮廓。
    ridge - 定义 3D 凸槽轮廓。
    inset - 定义 3D 凹边轮廓。
    outset - 定义 3D 凸边轮廓。
    none - 定义无轮廓。
    hidden - 定义隐藏的轮廓。
    outline-width 指定轮廓的宽度 
                    p.ex1 {
                    border: 1px solid black;
                    outline-style: solid;
                    outline-color: red;
                    outline-width: thin;
                  }
    		thin（通常为 1px） medium（通常为 3px） thick （通常为 5px） 特定尺寸（以 px、pt、cm、em 计）
    outline-color 设置轮廓的颜色。 
                    p.ex1 {
                    border: 2px solid black;
                    outline-style: solid;
                    outline-color: red;
                  }
    		也可使用outline-color: invert。执行颜色反转。可以确保无论颜色背景如何，轮廓都是可见的
    outline 简写属性 
                    p.ex3 {outline: 5px solid yellow;}
    		outline-width outline-style（必需） outline-color
    outline-offset 在元素的轮廓与边框之间添加空间。元素及其轮廓之间的空间是透明的。 
                    p {

    CSS overflow [溢出]

    CSS overflow 属性控制对太大而区域无法容纳的内容的处理方式。
    可设置以下值：
    • visible - 默认。溢出没有被剪裁。内容在元素框外渲染
    • hidden - 溢出被剪裁，其余内容将不可见
    • scroll - 溢出被剪裁，同时添加滚动条以查看其余内容
    • auto - 与 scroll 类似，但仅在必要时添加滚动条
    Attribute 说明 实例 Tips
    overflow: visible 定在元素的内容太大而无法放入指定区域时是剪裁内容还是添加滚动条。
    overflow: hidden 如果使用 hidden 值，溢出会被裁剪，其余内容被隐藏
    overflow: scroll 如果将值设置为 scroll，溢出将被裁剪，并添加滚动条以便在框内滚动。请注意，这将在水平和垂直方向上添加一个滚动条（即使您不需要它）
    overflow: auto auto 值类似于 scroll，但是它仅在必要时添加滚动条
    overflow-x 和 overflow-y overflow-x 指定如何处理内容的左/右边缘。
    overflow-y 指定如何处理内容的上/下边缘。    		 
                    div {
                    overflow-x: hidden; /* 隐藏水平滚动栏 */
                    overflow-y: scroll; /* 添加垂直滚动栏 */
                  }

    CSS float [浮动] 与 清除浮动

    float 属性用于创建浮动框，将其移往一边，知道某一边缘接触及 包含块 或 另一浮动框 的边缘。 Float（浮动），会使元素向左或向右移动，其周围的元素也会重新排列。
    往往是用于图像，但它在布局时一样非常有用。

    float 属性可以设置以下值之一：
    • left - 元素浮动到其容器的左侧
    • right - 元素浮动在其容器的右侧
    • none - 元素不会浮动（将显示在文本中刚出现的位置）。默认值。
    • inherit - 元素继承其父级的 float 值

    为何需要浮动

    1、行内块元素可实现一行显示，但之间会有大的空白间隙，很难控制，不满足实际开发需求。
    2、很多布局方式，标准流无法完成，此时利用 浮动 完成布局。因为浮动可以改变元素标签默认的排列方式。

    浮动特性

    1、浮动元素会脱离标准流 （脱标），脱标后不再保留原先的位置 （上浮，z轴+1）
    2、浮动元素会一行内显示，并且元素顶部对齐
    3、浮动元素是互相贴合（无间隙），如果父级元素装不下这些浮动盒子，多出的盒子会另起一行
    4、浮动元素会具有 行内块元素 的特性
    5、浮动盒子只会影响浮动盒子后边的标准流，对前面标准流无影响

    浮动元素常和标准流父级元素搭配使用

    父级元素约束浮动元素位置：
         先用标准流 父级元素 排列 上下 位置，之后 内部元素 采取浮动排列 左右 位置。

    为什么要清除浮动

    1、父盒子不一定有高度，很多时候，是让子盒子撑开父盒子
    2、浮动元素不占有高度，会对后面元素排版产生影响

    清除浮动本质

    1、本质是清除浮动元素造成的影响
    2、如果父元素有高度，则不需要清除浮动
    3、清除浮动后，父盒子会根据子盒子自动检测高度。父级有了高度，就不会影响后面的标准流了。

    clear 属性

    clear 属性可设置以下值之一：
    • none - 允许两侧都有浮动元素。默认值
    • left - 左侧不允许浮动元素
    • right- 右侧不允许浮动元素
    • both - 左侧或右侧均不允许浮动元素
    • inherit - 元素继承其父级的 clear 值
    实际工作中：几乎只用 clear：both；
    清除浮动的策略是：闭合浮动（只让浮动在父盒子内部影响）

    清除浮动方法

    1、额外标签法（隔墙法）：W3C推荐方法
    2、父级添加 overflow 属性
    3、父级添加 after 伪元素
    4、父级添加双伪元素

    缺点：添加许多无意义的标签，结构化较差
    额外标签法 在浮动元素末尾添加空标签（必须是块级元素） 
                        <div style="clear:both"></div>
                    或
                    <br style="clear:both" />
    		优点：通俗易懂，易于书写
    父级添加 overflow 给父级添加 overflow 属性，将其属性设置为hidden、auto、sscroll 
                        .box {
                        overflow: hidden;
                        width: 1200px;
                        background-color: aquamarine;
                    }
    		说明：如果一个元素比父元素高，并且它是浮动的，它将“溢出”到其容器之外。
    然后我们可以向包含元素添加 overflow: auto;，来解决此问题

    优点：代码简洁
    缺点：无法显示溢出部分
    父级添加 after 伪元素 相当于 额外标签法 的升级版 
                        /* 设置空的h=0块级元素，清除浮动，隐藏 */
                    .clearfix::after {
                        content: "";
                        display: block;
                        height: 0;
                        clear: both;
                        visibility: hidden;
                    }
                    /* IE 6、7 专有 */
                    .clearfix::after {
                        *zoom: 1;
                    }
    		实际常用，可复制代码标签调用 优点：没有增加额外标签，结构更简单
    缺点：需要照顾低版本浏览器
    父级添加双伪元素 在 after 的基础上 在父级元素 before 添加 
                        .clearfix,
                    .clearfix::after {
                        content: "";
                        display: block;
                        height: 0;
                        clear: both;
                        visibility: hidden;
                    }
                    /* IE 6、7 专有 */
                    .clearfix::after {
                        *zoom: 1;
                    }
    		优点：代码更简洁
    缺点：要照顾低版本浏览器

    网页布局

    传统网页布局的三种方式

    • 标准流（普通流）
      • 标签按默认方式排列，最基本的布局方式。
    • 浮动
    • 定位
    注意：实际开发中，一个页面都基本包含了这三种布局方式。（移动端有新布局方式）

    网页布局准则

    1、多个块级元素 纵向 排列找标准流， 横向 排列找浮动。
    2、先设置盒子大小，再设置盒子位置。

    常见网页布局

    网页布局有很多种方式，一般分为以下几个部分：页眉、菜单、内容和页脚。

    【 页眉（header） 】 通常位于网站顶部（或顶部导航菜单的正下方）。它通常包含徽标（logo）或网站名称

    【导航栏】 包含链接列表，以帮助访问者浏览您的网站

    【内容】 一般有三种形式:
    • 1 列：一般用于移动端
    • 2 列：一般用于平板设备
    • 3 列：一般用于 PC 桌面设备
    不相等的栏
    主要内容（main content）是您网站上最大、最重要的部分。

    1、列宽不相等的情况很常见，因为大部分空间都为主内容保留。
    2、附属内容（如果有）通常用作替代导航或指定与主要内容有关的信息。
    3、您可以随意更改宽度，只要记住它的总和应为 100％

    【 页脚 】位于页面底部。它通常包含诸如版权和联系方式之类的信息。

    响应式网站布局

    创建一个自适应的网站布局，会根据屏幕宽度切换两列与全宽列

    页面布局整体思路

    • 通过测量确定页面版心（可视区）
    • 分析行模块，行模块中的列模块
    • 行中列模块 浮动 布局，确定列大小、位置
    • 制作html结构。先结构后样式

    网页布局总结

    一个完整的网页，是标准流、浮动、定位一起完成布局的，每个都有自己的用法
    1、标准流
    可以让盒子上下排列或者左右排列，垂直的块级盒子显示就用标准流布局
    2、浮动
    可以让多个块级元素一行显示，多个块级盒子水平显示就用浮动布局
    3、定位
    可以让多个盒子前后叠压显示，元素自由在某个盒子内移动就用定位布局

    元素的显示与隐藏 [display visibility overflow]

    display 属性

    • display: none ; 隐藏对象 【用处更多】
    • display: block ; 除转换块级元素外，同时还有显示对象的意思
    display 隐藏元素后，不再占有原来的位置
    后面应用广泛，搭配 js 可以做很多网页效果

    visibility 属性

    • visibility: hidden ; 元素隐藏
    • visibility: visible ; 元素可见
    visibility 隐藏元素后，继续占有原来的位置

    overflow 属性

    • overflow: visible ; 不剪切内容也不添加滚动条
    • overflow: hidden ; 不显示超出对象尺寸的内容，超出的部分隐藏掉
    • overflow: scroll ; 不管是否超出对象，总显示滚动条
    • overflow: auto ; 超出对象显示滚动条，不超出对象不显示滚动条
    一般情况，不让溢出的内容显示出来，会影响布局
    但是有定位的盒子（超出父盒子），请慎用 overflow:hidden ，因为它会隐藏多余的部分

    精灵图

    目的: 为有效地减少服务器接收和发送请求的次数，提高页面的加载速度，出现了CSS精灵技术

    核心原理：将网页中的一些小背景图整合到一张大图中去，这样服务器只需要一次请求即可

    精灵图的使用
    1、 主要针对背景图片使用，就是把多张小背景图片整合到一张图片上 2、 这个大图片也称为 sprites 精灵图或雪碧图

    CSS 初始化

    重设浏览器的样式，也称为CSS reset，每个网页都必须首先进行CSS初始化。
